# Commercial Use and Business Guidance

## The Goal: "Let It Spread, But Don't Clone and Compete"

FishTankController is licensed under Apache License 2.0, which **permits commercial use**. This document clarifies what that means in practice and outlines community expectations for commercial activities.

## What the Apache License Allows

The Apache License 2.0 grants you the legal right to:
- Use the software commercially without paying royalties
- Modify the software for commercial purposes
- Distribute the software as part of commercial products or services
- Sell products or services that incorporate this software

**Important**: These are legal permissions. The guidelines below are community expectations that complement (not override) the license.

The following commercial uses are **explicitly encouraged and welcome**:

- Charging for installation services at homes, businesses, or aquarium shops
- Professional setup, configuration, and calibration services

- Selling hardware kits with FishTankController firmware pre-installed

- Include a copy of the LICENSE file with the hardware
- Provide attribution (e.g., "Powered by FishTankController")
- Make clear where users can get source code and updates
- Comply with Apache License 2.0 requirements

### Consulting and Custom Development
- Paid consulting for FishTankController implementations
- Custom feature development for specific clients
- Integration services with other aquarium or home automation systems
- Training and education services

### SaaS and Cloud Services
- Offering cloud monitoring services that work with FishTankController
- Data logging and analytics services
- Remote management platforms
- Backup and synchronization services

## ‚ö†Ô∏è Discouraged Activities (Trademark and Branding Considerations)

While legally permitted under Apache 2.0, the following activities conflict with trademark rights and community expectations:

### Standalone Product Resale Using Original Branding
- **Scenario**: Taking the unmodified or minimally modified software, packaging it as your own product, and selling it as "FishTankController" or a confusingly similar name
- **Issue**: Creates user confusion about the source, support, and official nature of the product
- **Guidance**: If you want to sell FishTankController as a standalone product, consider:
  - Contacting the project maintainer about becoming an official partner/reseller
  - Creating a distinct fork with your own branding (see [TRADEMARK.md](TRADEMARK.md))
  - Ensuring you add significant value (support, warranty, custom hardware, etc.)

### Competing Products with Original Branding
- **Scenario**: Creating a competing commercial aquarium controller product that uses "FishTankController" branding without permission
- **Issue**: Violates trademark rights (separate from code license)
- **Guidance**: You're free to build competing products using the code, but use your own branding

## üìã Best Practices for Commercial Use

To be a good member of the FishTankController community while running a commercial operation:

1. **Maintain Attribution**
   - Always credit the original project
   - Include LICENSE and NOTICE files
   - Don't remove copyright notices from source code

2. **Contribute Back** (Encouraged, Not Required)
   - Submit bug fixes and improvements upstream
   - Share documentation and tutorials
   - Sponsor the project or maintainers if possible

3. **Provide Clear Support Boundaries**
   - Make clear whether support comes from you or the upstream project
   - Don't direct your customers to the main project's issue tracker for paid product support

4. **Be Transparent**
   - Clearly state that your product is based on FishTankController
   - Don't misrepresent your relationship with the project
   - If you modify the software, make that clear to your customers

5. **Follow the Apache License Requirements**
   - Include NOTICE file if you distribute the software
   - Provide LICENSE file with distributions
   - State changes you made to the original software
   - Don't use the FishTankController trademark without permission (see [TRADEMARK.md](TRADEMARK.md))
